Реализациия семейства функции tioGet* и tioGetDef*
1. Как работают функций tioGetL/tioGetD/tioGetC
а) Функция long tioGetL(const char* name) получает по имени параметра 
	 значение, которое либо конвертируется в тип long при возможности,
	 либо сообщает об ошибки TEINCTYPE (невозможности привести к запрошенном типу).
	 Также возможны другие ошибки, описанные в спецификации на прототипы функций.
	 Функция конвертирует в тип long только те данные, которые по своей структуре
	 являются целым числом и не переполняют тип long.
	 Для десятичной системы счисления это - цифры [0-9]. Остальные символы являются
	 недопустимыми.
	 Пример:
		Допустимые значения: 
			128382
			093923
			-83929
		Недопустимые значения:
			0xFF
			12345e2
			12345E2
			0.12345
б) Аналогичным образом работают функции tioGetC/tioGetDefC/tioGetDefL
в) Функция double tioGetD(const char* name) работает схожим образом с tioGetL. Однако следует учитывать
возможную потерю точности при извлечении данных из значения ключа. Которая может зависеть от архитектуры и 
операционной системы.
